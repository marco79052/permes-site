1. What is Permes

Permes has two components, and both must stay running:

Component Role
Permes Hub Launcher & manager: license activation, executor download/updates, recharge, points & events, customer support
Permes Executor The worker: reads the game screen (OCR + pixel recognition) and sends keys according to the rotation rules

⚠ Keep Permes Hub running while in use (a persistent banner on the executor window says the same). Closing the Hub stops the executor.

Left: Permes Hub main window (Launch Permes, Recharge, Settings, Points Ranking, Invite & Earn, Points Lottery, Mini Games). Right: Permes Executor console (Start/Stop and the Install Addon / Calibrate / Train Model / Rotation Rules / Survival/Heal Settings / Switch Module / SimC Sim tabs).

2. Quick Start (new users)

Full flow: install Hub → activate → launch the executor → install the in-game addon → in-game setup → calibrate → (optional) model training → fight.

2.1 Install Permes Hub

  1. Run the installer PermesHub_Setup.exe.
  2. The wizard notes: the install path and program name are automatically disguised — no manual setup needed. This is intentional, not malware.
  3. Pick a drive, optionally check "Create desktop shortcut" / "Run at startup" / "Run after install", then click Install.

2.2 Activate & Sign In

  1. Open the Hub and enter your purchased license key.
  2. Once activated, the Hub shows your modules, expiry date (or "Lifetime access"), and points balance.
  3. The key is bound to your device. Changing computers requires unbinding (see FAQ).

2.3 Launch the Permes Executor

  1. Click the big green Launch Permes button in the Hub.
  2. On first launch the executor downloads automatically — wait for the progress bar.
  3. The executor console appears with a row of function tabs:
    Install Addon | Calibrate | Train Model | Rotation Rules | Survival/Heal Settings | Switch Module | (Hotkeys) | SimC Sim.

If you use a Logitech mouse, the Hub's GHUB Key Protection button opens Logitech G HUB to work with key protection.

2.4 Install the In-Game Addon

  1. In the executor, click Install Addon — the "Select Classes to Install" dialog appears.
  2. Make sure the target directory points to the retail _retail_ folder (e.g. D:\World of Warcraft\_retail_) — this is the most common mistake; the addon won't work in the wrong folder.
  3. Check the classes you play (all checked by default), then click OK.

After installation the addon shows up in the in-game AddOn list under a randomized name (e.g. P3F8847DE). This disguise is intentional — not finding anything called "Permes" is normal.

2.5 Switch Module (choose class & spec)

The first thing to do after entering the game: click Switch Module in the executor, pick your current character's class + spec, then OK.

⚠ Pick the spec of the character you are actually playing. The wrong spec means wrong rotation, wrong keybinds, and wrong recognition regions.

3. First-Time In-Game Setup

3.1 Enable the Addon

  1. Press ESC → AddOns.
  2. Check Load out of date AddOns at the top.
  3. Make sure the Permes addon (random name, e.g. P3F8847DE) is checked; if anything shows red "Requires Reload", click Reload UI.


3.2 Set Up the Cooldown Manager

Permes reads ability cooldowns and buff stacks through the game's built-in Cooldown Manager, which must be configured correctly:

  1. Press ESC → Edit Mode, and enable the Cooldown Manager frames.
  2. Select the Essential Cooldowns frame; recommended settings:
  3. Orientation: Horizontal; Icon Direction: Right
  4. Icon Size ≥ 110% (120%+ recommended) — icons that are too small can't be recognized
  5. Visibility: Always Visible; check Show Timer / Show Tooltips
  6. Select the Tracked Buffs frame with the same settings, plus:
  7. ⚠ "Hide When Inactive" must be UNCHECKED, otherwise Permes cannot read the buff numbers.


3.3 Import the Official Layout

If your Cooldown Manager layout doesn't match the official requirement (a banner appears saying required abilities are missing):

  1. Don't rearrange the frames by hand — click Official Layout on the Permes dock (or "Import official layout" on the banner).
  2. The game reloads the UI automatically.
  3. ⚠ After importing, do not move the icons Permes reads — pixel recognition samples fixed positions; moving them breaks recognition.

3.4 Calibrate

Calibration tells the executor where each number sits on your screen:

  1. Click Calibrate in the executor (or "Calibrate" on the dock).
  2. The chat log prints: 5 ROI coordinates exported, please /reload.
  3. Type /reload in chat — calibration only takes effect after a reload.
  4. The dock status changes from "Calibrate: NotDone" to Done — success.

Verify calibration:

  1. Click ▶ Start in the executor and hit a training dummy.
  2. Watch the executor's OCR Recognition values (e.g. wound stacks, ability stacks) — they should match the numbers on the in-game buff icons in real time.
  3. Match = calibrated. Mismatch = go to 9. Troubleshooting.

4. AI Model Training (required when recognition is off)

Permes' number-recognition model is trained and runs locally. You must train your own model when:

  • Using Permes for the first time (your resolution / DPI / UI scale differs from the default environment)
  • After changing PC, monitor, resolution, or in-game UI scale
  • Calibration succeeded but OCR numbers are still wrong

Training steps

  1. Click Train Model in the executor.
  2. If told an existing fine-tuned model was found:
  3. Never trained before → choose full manual retrain
  4. Have a model and want to improve it → choose auto/manual incremental training
  5. Auto capture begins: the program screenshots every 2 seconds, 30 shots total. During capture:
  6. Stay at a training dummy and keep fighting;
  7. Make sure the Cooldown Manager icons and buff numbers keep changing (those numbers are the training data);
  8. Keep the Master switch on so the rotation fights for you.
  9. After capture comes data labeling: for first-time use, manual annotation is recommended ("AI auto annotation" is Beta).
  10. Review each of the 30 shots: fix wrong values directly in the input box; leave regions with no number empty.
  11. ⚠ One ROI box must never cover two numbers — the AI would read them as a two-digit number. Fix the layout first if that happens.
  12. Training then starts, and the executor restarts automatically when done — this is normal.

5. Daily Use

5.1 The In-Game Dock

The vertical floating menu on the left of the game screen is your daily control center:

Item Function
Master Green = on / red = off. Permes sends keys per the rotation while on. Keep it on
Burst Burst ability switch — on during burst phases
Mode Auto / Single-target / AoE
Settings Opens the addon settings panel (see 5.2)
Hidden Hides the dock
Calibrate Shows calibration status; click to recalibrate
Official Layout One-click import of the official UI layout

5.2 Addon Settings Panel (dock → Settings)

① Settings tab

  • Keybinds for Master / Burst / Mode toggle (Master and Burst default to mouse side buttons BUTTON4/BUTTON5)
  • Checkboxes: Auto target, Debug info, Disable text, Disable sound
  • Bottom: SimC Import (see 6.4), Reset

② Import Talents tab

Lists official talent builds (e.g. a 1-minute burst build and a 30-second burst build). Select one and click Apply.

③ Dedicated Key Macros tab

Every ability of the current spec with its keybind; click Edit next to an ability to change it. These keybinds must match your actual in-game keybinds for Permes to press the right keys.

④ CVarSettings tab (game parameter optimization)

  • SpellQueueWindow: recommended 150
  • The parameter table shows Current / Target / Status per row — all green "Safe" means OK
  • Just click Apply Recommended Settings for a one-click fix

⑤ Create Key Macros tab

General macro commands — type them in chat to apply, or copy them into an in-game macro and bind to a key:

Command Effect
/ON Master switch on
/OFF Master switch off
/bfk Burst on
/bfg Burst off
/dan Single-target mode
/qun AoE mode
/zi Auto mode
/charu Temporarily disables Master for 0.5s (for weaving in manual abilities)

Typical /charu usage — weave mouseover manual abilities into a macro so they don't fight the auto rotation:

/charu /cast [@mouseover,harm,exists]Death Grip;Death Grip

/charu /cast [@player]Anti-Magic Zone

6. Advanced Features

6.1 Rotation Rules

Executor → Rotation Rules: the left side lists action sources (ability → keybind), the right side is the rotation pipeline:

  • Rules execute top-down by priority; each rule can carry conditions (e.g. "runes ≥ 4", "target distance ≤ 5", "own HP ≤ 30").
  • Regular users: just click "Import" and pick an official rule set — no editing needed.
  • Advanced users: reorder, add/remove rules, attach conditions; click Save to current rule when done; rules support import/export for local backup.

Example: adding the condition own HP ≤ 30% and runic power ≥ 30 to Death Strike gives you automatic low-HP healing.

6.2 Survival / Healing Settings

Executor → Survival/Heal Settings. Everything is off by default; options only take effect when checked:

  • Auto interrupt: interrupt current target / focus target, with configurable delay (1 loop = 500ms; ~0.25s recommended to look human and avoid instant interrupts)
  • Defensive/consumable thresholds: cast automatically when HP drops below the set percentage (e.g. Anti-Magic Shell ≤60%, Icebound Fortitude ≤35%, HP pot ≤50%, Healthstone ≤30%)
  • Healing settings: automatically focus critically injured party members via Pixel Bar health (tanks are auto-skipped); critical threshold defaults to HP below 20%

Click Save when done.

6.3 Hotkeys

Executor → hotkey settings: global start hotkey (default CTRL+P, starts rotation + burst) and stop hotkey (default CTRL+O).

6.4 SimC

  1. Download SimulationCraft (SimC).
  2. In the addon settings panel → Settings tab → SimC Import.
  3. After exporting the SimC string, run /reload in-game to save it to SavedVariables.

8. Pre-Flight Environment Checklist

Wrong recognition or a broken rotation? Run through this list first — 90% of issues live here:

  • [ ] In-game render scale = 100% (Options → System → Graphics)
  • [ ] GPU control panel render scale also = 100% (NVIDIA app → program settings → WoW — both places must agree)
  • [ ] Windows display scaling = 100%, resolution at the recommended value (Settings → System → Display)
  • [ ] Multi-monitor: check scaling and resolution on every screen
  • [ ] Cooldown Manager enabled, icon size ≥ 110% (120%+ recommended)
  • [ ] "Hide When Inactive" unchecked on Tracked Buffs
  • [ ] No UI windows covering the monitored areas (character pane, bags, etc. — Permes can't "see" covered pixels)
  • [ ] CVarSettings all green "Safe" (when in doubt, click "Apply Recommended Settings")
  • [ ] Dock shows Calibrate: Done
  • [ ] Switch Module matches your current character's spec

9. Troubleshooting

9.1 Self-Check: Game Status vs OCR

Open the executor window and compare two areas with the actual game state:

  • Game Status: Master, Burst, runes, runic power, target HP, etc. — do they match the game?
  • OCR Recognition: while hitting a dummy, do the recognized buff stacks/cooldown numbers match the in-game icons one by one?

Whichever area mismatches tells you where the problem is: Game Status wrong → calibration/layout issue; OCR numbers wrong → calibration or model issue.

9.2 Standard Recalibration

  1. Click Stop in the executor.
  2. Click Calibrate on the dock (or in the executor).
  3. When the chat log shows 5 ROI coordinates exported, run /reload.
  4. Confirm the dock shows Calibrate: Done again, and drag any moved tracking icons back into place.
  5. Click Start and re-verify on a dummy.

9.3 "Stuck Ability" Diagnosis

An ability is off cooldown but never fires:

  1. Open Rotation Rules, find that ability's rule, and check whether its conditions are miswritten (a condition that is never true stalls the rule).
  2. Verification (bisect):
  3. Delete all conditions from the suspect rule, or add an unconditional copy of it;
  4. If the unconditional rule fires non-stop → the original rule's conditions are the problem;
  5. Delete the temporary rule when done.

9.4 Manual Calibration & Model Issues

  1. Executor → Advanced Settings (red banner "⚠ Don't touch if unsure") → Manual Calibration.
  2. If OCR is still wrong after manual calibration, it's a model problem → go back to 4. AI Model Training.

9.5 Common Errors Quick Reference

Symptom Cause & fix
Addon installed but invisible in-game Install directory didn't point to _retail_; the addon uses a random disguised name (e.g. P3F8847DE), not "Permes"
Addon inactive after entering game Check "Load out of date AddOns" in the AddOn list and Reload UI
OCR shows -- or garbage Run the checklist, then recalibrate, then retrain the model
Two-digit misreads An ROI box covers two numbers — fix the layout so each tracked region holds exactly one number
Buff numbers unreadable "Hide When Inactive" is checked on Tracked Buffs — uncheck it
Everything suddenly unrecognized Cooldown icons were moved or UI scale changed — re-import the official layout and recalibrate
Executor restarts after training Normal behavior
Executor warns to keep Hub running The Hub was closed — reopen it

10. FAQ

Compiled from the Help & Diagnosis knowledge base, grouped by scenario.

License & Activation

Q: "Machine ID mismatch"?
Caused by changing computers or major hardware. Contact support to verify and unbind the old device, then re-enter your original key on the current one.

Q: "Device binding limit reached" on a new PC?
Just activate on the new computer — the oldest device is unbound automatically. To keep a specific old device, sign out on the others first. Still failing? Ask support to clear old devices.

Q: "License key not found / already used"?
Check for typos (extra spaces, mixed-up letters/digits) and make sure the key came from an official channel. If it's still reported missing, contact support to check the key status.

Q: "Expired / revoked / pending approval / device banned"?
Expired → renew in the Hub or buy a new key. Pending approval (promo/referral keys) → ask support or your inviter to speed it up. Revoked/banned → contact support for the reason.

Q: Hub not activated / no active license?
Re-enter your key in the Hub to refresh, and update the Hub to the latest version.

Q: Session expired / signed out?
Check the network and restart the Hub; make sure the same key isn't running on another device (multi-login kicks you out).

Q: Is there a trial?
One trial per device and per network; the trial program is currently paused — please use a full license key.

Network & Updates

Q: "Invalid signature/timestamp", or everything fails at once?
Set Windows time to sync automatically (Settings → Time & language), turn off any proxy/VPN, then restart the Hub.

Q: "Cannot connect to server / request timed out"?
Confirm you can browse the web; disable proxy/VPN; whitelist the Hub in firewall/antivirus and restart. "Under maintenance" → retry in a few minutes.

Q: "Too many requests"?
Wait about a minute; on shared networks (office/internet cafe), switch to a mobile hotspot.

Q: Download failed / package verification failed?
Retry so the Hub fetches a fresh link; switch networks (e.g. hotspot) to avoid hijacking; if verification keeps failing, note the error code in brackets and send it via Help & Diagnosis.

Payment

Q: Cannot create a payment order?
Check for unpaid pending orders (they occupy your order quota and expire in ~30 minutes); lifetime keys don't need renewal; repeated failures → contact support.

Q: Paid but nothing arrived?
First confirm the charge in your Alipay/PayPal bill. Charged → wait on the payment page for confirmation, usually a few minutes — do not pay again; over 30 minutes → contact support with your order number. Not charged → simply reorder.

Q: Primary Alipay failed, backup channel prompts?
If the primary channel fails, follow the page prompt to use backup Alipay. If the primary order is already paid, go back to the Hub and wait — do not pay again via backup. If the status can't be confirmed, check your bill first and never pay twice.

Q: PayPal failed/cancelled?
Cancelled or failed → reorder in the Hub. Paid → wait for confirmation in the Hub, don't pay again.

Modules & Specs

Q: "No permission for this spec" or spec greyed out?
Not purchased → add the spec in the Hub and restart Permes. Just purchased → restart the Hub to refresh permissions. "Module retired" → update Permes and pick another module; old calibration data is kept.

Events & Points

Q: Season card flash sale issues?
"Busy" → retry immediately a few times; sold out/not started → wait for the next sale; "already secured" → no further action needed.

Q: Insufficient points / unlock failed?
Earn points via referrals and events; if points were deducted but the reward didn't arrive, contact support.

Q: Cannot create a referral?
Your key needs ≥ 15 days remaining — renew first. Invite keys only work after support approval.

Q: AI Support limited?
Up to 5 questions per hour — wait an hour if you hit the cap; include the full error message and steps in one question.

Q: Name change failed?
The leaderboard name can only be changed once every 30 days.
